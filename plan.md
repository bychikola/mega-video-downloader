# План: Сайт для скачивания видео с YouTube

## Контекст

Создание веб-приложения для скачивания видео с YouTube. Пользователь вставляет ссылку, видит информацию о видео, выбирает формат (MP4/MP3/WebM) и качество, скачивает файл. Проект с нуля.

## Стек

| Слой | Технология |
|------|-----------|
| Фреймворк | Next.js 14+ (App Router) |
| Стилизация | Tailwind CSS |
| Иконки | Lucide Icons |
| Уведомления | Sonner (тосты) |
| Видео-бэкенд | yt-dlp (CLI) |
| Конвертация | ffmpeg |
| Деплой | VPS Ubuntu + PM2 + Nginx |

## Архитектура

Единое Next.js-приложение на VPS. API Routes вызывают yt-dlp как дочерний процесс для получения метаданных и скачивания. ffmpeg для конвертации форматов.

```
Browser → Next.js (React Frontend) → /api/info, /api/download → yt-dlp + ffmpeg
```

## API

### `POST /api/info` — метаданные видео
- Принимает: `{ "url": "..." }`
- Возвращает: `{ id, title, duration, thumbnail, formats: [{ id, quality, ext, size, note }] }`
- Реализация: `yt-dlp --dump-json <url>` → парсинг JSON → фильтрация лучших форматов

### `POST /api/download` — скачивание
- Принимает: `{ "url", "formatId", "ext" }`
- Возвращает: бинарный поток файла с Content-Disposition
- Реализация: yt-dlp скачивает во временную папку → ffmpeg при необходимости → стриминг клиенту → удаление временного файла

## UI (3 состояния)

1. **Начальный экран**: заголовок, поле ввода URL, кнопка (неактивна при невалидной ссылке)
2. **Загрузка**: спиннер, пока yt-dlp получает данные
3. **Результат**: карточка видео (превью + название + длительность), табы форматов (MP4/MP3/WebM), сетка качеств с размерами, кнопка скачивания

## Обработка ошибок
- Невалидная ссылка → красная подсветка поля
- Видео недоступно → карточка с пояснением
- Ошибка сервера → toast с предложением повторить
- Обрыв загрузки → кнопка «Повторить»

## Порядок реализации

1. Инициализация Next.js проекта с TypeScript и Tailwind
2. Типы данных (`types/video.ts`)
3. Библиотека для работы с yt-dlp (`lib/yt-dlp.ts`) и фильтрация форматов
4. API-роуты: `/api/info` и `/api/download`
5. Компоненты: `VideoInput` → `VideoCard` → `FormatSelector` → `DownloadButton` → `ErrorMessage`
6. Главная страница — сборка всех компонентов с управлением состоянием
7. Стилизация и анимации
8. Обработка ошибок и граничных случаев
9. Очистка временных файлов (`lib/cleanup.ts`)
10. Тестирование на реальных ссылках

## Верификация
- Вставить ссылку на публичное YouTube видео → проверить отображение метаданных
- Выбрать разные форматы/качества → проверить корректность форматов
- Скачать видео → проверить файл открывается и соответствует качеству
- Вставить ссылку на приватное/удалённое видео → проверить сообщение об ошибке
- Вставить невалидную строку → проверить валидацию на клиенте
